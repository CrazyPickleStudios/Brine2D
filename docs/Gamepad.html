Gamepad input

Brine2D exposes gamepads via IGamepads (manager) and IGamepad (device). You can query buttons, per-frame edges, and analog axes for multiple controllers. The desktop SDL host supports hot-plug, remapping, and rumble where available.

Quick start

// Access the primary gamepad (first connected)
var pads = context.Gamepads;
var pad = pads.Primary;

if (pad != null && pad.IsConnected)
{
    // Left stick movement (normalized, with deadzone applied)
    var move = pad.LeftStick; // X: -1..+1, Y: -1..+1

    // Button edges (fire on the frame they change)
    if (pad.WasButtonPressed(GamepadButton.A))
        Shoot();

    // Right shoulder held
    if (pad.IsButtonDown(GamepadButton.RightShoulder))
        FireHeld();

    // Triggers as analog (0..1)
    float throttle = pad.RightTrigger;
}

Core API

  • IGamepads.Count, IGamepads.Primary, IGamepads.Get(i)
  • IGamepad.IsConnected, Name
  • Buttons: IsButtonDown/WasButtonPressed/WasButtonReleased(GamepadButton)
  • Axes:
    • Vector2 LeftStick, Vector2 RightStick — normalized -1..+1 per component.
    • float LeftTrigger, float RightTrigger — normalized 0..1.
    • float GetAxis(GamepadAxis) — direct axis read.
  • Deadzones and feel:
    • float StickDeadzone (default 0.15)
    • bool RadialDeadzone (default true) — radial with rescaling; set false for per-axis deadzones
    • Axis inversion per stick: InvertLeftX/Y, InvertRightX/Y
  • Trigger edges with hysteresis:
    • float TriggerPressThreshold (default 0.5)
    • float TriggerReleaseThreshold (default 0.45)
    • WasLeftTriggerPressed/WasLeftTriggerReleased, WasRightTriggerPressed/Released
  • Rumble (capability-based):
    • bool SupportsRumble, bool SupportsTriggerRumble
    • TryRumble(ushort low, ushort high, uint ms) — main motors
    • TryRumbleTriggers(ushort left, ushort right, uint ms) — trigger motors
    • StopRumble(), StopRumbleTriggers()

Examples

Movement and actions

var pad = context.Gamepads.Primary;
if (pad == null || !pad.IsConnected) return;

var move = pad.LeftStick; // (-1..1, -1..1)
MovePlayer(move);

if (pad.WasButtonPressed(GamepadButton.A))
    Jump();

var aim = pad.RightStick;
if (pad.IsButtonDown(GamepadButton.RightShoulder))
    Fire(aim);

Trigger edges with hysteresis

var pad = context.Gamepads.Primary;
if (pad == null) return;

// Optional: tune thresholds (defaults 0.5/0.45)
pad.TriggerPressThreshold = 0.6f;
pad.TriggerReleaseThreshold = 0.5f;

if (pad.WasRightTriggerPressed())
    StartCharging();

if (pad.WasRightTriggerReleased())
    ReleaseCharge();

Axis inversion (typical “up is negative” Y)

var pad = context.Gamepads.Primary;
if (pad != null)
{
    pad.InvertRightY = true; // if your camera expects +Y up
}

Rumble with capability checks

var pad = context.Gamepads.Primary;
if (pad != null)
{
    if (pad.SupportsRumble)
        pad.TryRumble(20000, 40000, 250);   // low/high strengths (0..65535), 250ms

    if (pad.SupportsTriggerRumble)
        pad.TryRumbleTriggers(30000, 0, 200); // left trigger kick
}

Behavior notes

  • Per-frame edges: queries like WasButtonPressed and trigger edge helpers are per-frame; call them during your Update.
  • Normalization: sticks are -1..+1; triggers are 0..1. Deadzones are applied by LeftStick/RightStick helpers.
  • Radial deadzone: by default, magnitudes inside the deadzone are zeroed and values outside are rescaled back to [0..1]. Disable with RadialDeadzone=false.
  • Focus changes: on focus loss, held state is cleared to prevent “stuck” input.
  • Hot-plug and remap: newly connected pads are available via IGamepads, and state is re-seeded on remap for consistent reads.

Advanced tips

  • Per-pad tuning: adjust StickDeadzone, RadialDeadzone and inversion flags per device to match your gameplay feel.
  • Trigger-as-button: use the trigger edge helpers for clean “press/release” semantics driven by analog thresholds.
  • Independent rumbles: main and trigger rumble are independent. Use StopRumble() and StopRumbleTriggers() separately when needed.
  • Axis access: when you need raw numbers, call GetAxis(GamepadAxis) directly (values are still normalized).

Troubleshooting

  • No input? Ensure you’re querying during Update. Check that a pad is connected (pads.Primary?.IsConnected).
  • Drifting sticks: increase StickDeadzone or switch to per-axis deadzones.
  • Rumble does nothing: check SupportsRumble/SupportsTriggerRumble. Some devices or platforms may not expose motors.
  • Trigger noise: adjust TriggerPressThreshold/TriggerReleaseThreshold to suit hardware.
  • Inverted Y expectation: set InvertLeftY/InvertRightY if your world treats +Y up.
