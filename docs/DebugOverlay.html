Debug overlay

The DebugOverlay provides on-screen performance and runtime diagnostics: FPS (smoothed + history), frame/update/draw timing, sprite batching stats, resource counts, managed heap usage, approximate GPU texture memory, window info, frame budget, and automatic severity coloring. It is lightweight, throttles text rebuilds, and supports customization (layout, colors, corner placement, alpha, scaling, font).

Quick start

// After creating SdlHost:
var host = new SdlHost();
// Overlay is created by SdlHost.Run() internally; default toggle: Ctrl/Cmd + F12.
// Inside your game Initialize you can customize:
host.ConfigureDebugOverlayTTF("fonts/JetBrainsMono-Regular@14.ttf"); // load a content font (optional)

// At runtime (e.g. in Update): toggle manually
if (host.WasShortcutPressed(Key.F11)) // custom key
{
    (host as SdlHost)?.Sprites; // overlay already hooked; or expose host.Overlay if you decide to.
}

// Change appearance (example):
// Corner: top-right, semi-transparent panel, show only text.
var overlay = typeof(SdlHost).GetField("_overlay", System.Reflection.BindingFlags.NonPublic | System.Reflection.BindingFlags.Instance)?.GetValue(host) as object;
// (Prefer adding a public accessor if needed.)

Core API

  • Visibility: Toggle(), SetVisible(bool)
  • Placement: SetCorner(OverlayCorner) (TopLeft / TopRight / BottomLeft / BottomRight)
  • Panel & border: SetPanelVisible(bool), SetBorderVisible(bool), SetBorderThickness(int), SetPadding(l,t,r,b), SetOuterMargin(int), SetPanelAlpha(byte), SetBorderAlpha(byte)
  • Colors: SetColors(text,panel,border), SetTextColor(Color), SetPanelColor(Color), SetBorderColor(Color)
  • Font: UseTtfFontFromContent(path,size), UseTtfFontFromMemory(byte[],size), UseTtfFontAsset(TtfFont), EnsureEmbeddedFallbackFont(int)
  • Wrapping: SetTtfWrapWidth(int), AutoWrapToWindow(int outerMargin), OnWindowResized()
  • Scaling (HiDPI/UI): SetUiScale(float) (applies to margins/padding/border)
  • Performance update cadence: SetUpdatePeriod(double seconds) (text rebuild throttle)
  • FPS target & severity coloring: SetTargetFps(double), SetAutoFpsColoring(bool)
  • Toggle chord: SetToggleChord(Key, KeyboardModifiers, bool usePlatformShortcut) (platform shortcut uses Ctrl on Windows/Linux, Cmd on macOS)
  • Metrics ingestion: UpdateMetrics(updateMs, drawMs) (called before Draw())
  • Rendering: Draw() (submits overlay sprites via shared sprite renderer)
  • Lifecycle: Dispose() (releases font texture & transfer buffer)
  • Query metrics: LastUpdateMs, LastDrawMs, LastFrameMs, SmoothedFps, LastSpriteItems, LastSpriteBatches, LastSpriteDrawCalls, TrackedResources

Displayed values

  • FPS: Exponential moving average + history (min/avg/max over last N samples)
  • Frame / Update / Draw times (milliseconds)
  • Frame budget: Derived from target FPS; over-budget frames flagged with "!"
  • Sprite stats: Items, batches, draw calls
  • Resources: Live and Total tracked, managed heap size, approximate GPU texture memory
  • Window & backend: Resolution, sRGB flag, active shader backend
  • Severity coloring: Text color shifts (normal/yellow/red) based on frame time thresholds

Examples

Basic customization

overlay.SetCorner(OverlayCorner.BottomRight);
overlay.SetPanelAlpha(200);
overlay.SetBorderVisible(false);
overlay.SetUiScale(1.25f); // enlarge spacing for HiDPI
overlay.SetTargetFps(120); // adjust frame budget
overlay.SetAutoFpsColoring(true);
overlay.SetUpdatePeriod(0.2); // refresh text 5 Hz

Custom toggle chord (disable platform shortcut)

// Use Shift+O (exact chord, no Ctrl/Cmd)
overlay.SetToggleChord(Key.O, KeyboardModifiers.Shift, usePlatformShortcut: false);

Applying a custom font

// Content pipeline must provide the font asset path (loader returns TtfFont)
overlay.UseTtfFontFromContent("fonts/JetBrainsMono-Regular@14.ttf");
overlay.AutoWrapToWindow(); // reflow text after font change

Manual wrap width (fixed column)

overlay.SetTtfWrapWidth(600); // fixed width instead of window adaptive

Color & transparency

overlay.SetColors(Color.White, new Color(32, 36, 40, 220), new Color(96, 100, 108, 255));
overlay.SetPanelAlpha(180);  // adjust opacity later without changing RGB

Integrating without auto creation (advanced)

// If you choose to manage it yourself:
var overlay = new DebugOverlay(host, (SdlSpriteRenderer)host.Sprites);
overlay.EnsureEmbeddedFallbackFont(14);
overlay.AutoWrapToWindow();
overlay.SetCorner(OverlayCorner.TopLeft);

// Each frame:
overlay.UpdateMetrics(updateMs, drawMs);
overlay.Draw();

Disabling severity coloring

overlay.SetAutoFpsColoring(false); // always use SetTextColor() value

Behavior notes

  • Throttled rebuild: Text surface regenerates at most every SetUpdatePeriod() interval or when metrics/layout/font change.
  • Resource memory: GPU texture bytes are approximate (width × height × format bytes). Extend if you add more formats.
  • Live vs Total: Total tracks all registrations over time; Live excludes disposed wrappers (overlay’s transient textures are untracked).
  • UI scale: Affects margins, border, padding; not font point size (reload font for true text scaling).
  • Severity thresholds: Red ≈ >125% of frame budget, Yellow ≈ >105%. Adjust logic in SeverityColor() if needed.
  • Safe font rebuild: Guards reduce risk of native crashes (font validity, wrap width cap, UTF-8 render path).
  • Draw order: Overlay batches with other sprites; ensure you call it before host Present().
  • Wrap width cap: Extremely large window widths clamp wrap width to avoid oversized surfaces.
  • Shadow effect: Provided by a second draw with offset; disable by modifying draw section if not desired.

Troubleshooting

  • Overlay not visible: Confirm SetVisible(true). Ensure Draw() is called after UpdateMetrics().
  • Text stale: Lower SetUpdatePeriod() or mark dirty via changing a setting (e.g. SetTargetFps()).
  • Resource count rising: Live should remain stable; Total counts historical registrations. Use Live for current usage.
  • Font render crash: Use embedded fallback (EnsureEmbeddedFallbackFont()) or verify the font wasn’t disposed elsewhere.
  • Colors not changing: Disable severity coloring (SetAutoFpsColoring(false)) if you want exact text color control.
  • Performance impact: Large wrap widths or very frequent update periods increase surface rebuild cost; raise period or reduce wrap width.

Extensibility ideas

  • Add GPU timing queries to display CPU vs GPU frame time.
  • Expose a public accessor for the overlay from SdlHost (e.g. property Overlay).
  • Integrate logging snapshot: method returning current overlay text for copying.
  • Custom severity palette or thresholds via strategy API.
