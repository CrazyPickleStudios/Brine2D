Mouse input

Brine2D exposes mouse input through the IMouse interface. It provides per-frame button edges, pointer position and movement, mouse wheel accumulation, cursor control, relative mode, and confinement. This page is a practical guide for using it in games and tools built with Brine2D.

Quick start

// Access the mouse from your engine context (e.g. inside Update/Draw)
var mouse = context.Mouse;

if (mouse.WasButtonPressed(MouseButton.Left))
{
    // Left clicked this frame
}

if (mouse.IsButtonDown(MouseButton.Right))
{
    // Right button is held
}

float x = mouse.X; // logical window coordinates (see HiDPI note below)
float y = mouse.Y;

Core API

  • Position and deltas (logical coords): X, Y, DeltaX, DeltaY
  • Wheel (per-frame accumulation): WheelX, WheelY
  • Buttons:
    • IsButtonDown(MouseButton)
    • WasButtonPressed(MouseButton) (edge: up → down this frame)
    • WasButtonReleased(MouseButton) (edge: down → up this frame)
  • Cursor / relative / capture / confine:
    • SetCursorVisible(bool), SetRelativeMouseMode(bool)
    • CaptureMouse(bool), SetConfinedToWindow(bool)
    • SetCursor(MouseCursor)
    • SetConfineRect(Rectangle? rect)
    • SetCursorPosition(int x, int y)
    • Queries: IsRelativeMouseModeEnabled, IsMouseCaptured
  • Double-click tuning:
    • SetDoubleClickTime(uint milliseconds)
    • SetDoubleClickRadius(float pixels)

Advanced features (desktop host)

Some hosts extend IMouse with additional functionality. When available, you can use:

  • HiDPI helpers: GetPixelPosition(), GetNormalizedPosition()
  • Drag detection: IsDragging(button), GetDragStart(button), GetDragDelta(button), DragThresholdPixels
  • Wheel tuning: WheelScale, InvertWheelX, InvertWheelY, WheelXScrolled, WheelYScrolled
  • Relative mode polish: RelativeSensitivity, EnableSmoothing, SmoothingFactor, FilteredDeltaX, FilteredDeltaY
  • Confinement helpers: SetAutoConfineToWindow(bool enable, int padding = 0)
  • Cursor management: PushHideCursor(), PopHideCursor(), CenterCursor(), TrySetCursorFromSurface(...)
  • State helpers: Snapshot(), DownMask, PressedMask, ReleasedMask, IsInWindow
  • Click metadata: LastClickButton, LastClickCount, LastClickTimeSeconds
  • Events: OnClick, OnDoubleClick, OnMultiClick(button, count), OnDragStart, OnDragEnd
  • Lifecycle: AutoHideCursorAfterSeconds (inactivity auto-hide), Reset() (clear all state)

Note: Advanced features depend on the active host. If a feature is not available, it’s a no-op or not exposed by the host’s mouse type.

Examples

Relative mouse mode

context.Mouse.SetRelativeMouseMode(true); // deltas update, pointer may be locked

Confine to a region

// Confine the cursor to a rectangle within the window (logical coords)
context.Mouse.SetConfinedToWindow(true);
context.Mouse.SetConfineRect(new Rectangle(100, 100, 400, 300));

Programmatic reposition and cursor shape

context.Mouse.SetCursor(MouseCursor.Hand);
context.Mouse.SetCursorPosition(200, 150);

Double-click behavior

// Configure multi-click detection window (set early, before you start handling input)
context.Mouse.SetDoubleClickTime(250);         // milliseconds
context.Mouse.SetDoubleClickRadius(4f);        // pixels

Behavior notes

  • Per-frame edges: WasButtonPressed/Released and wheel values are cleared each frame. Read them during your Update loop.
  • Logical vs pixel coordinates: X/Y are logical window units (DPI independent). If you render in back-buffer pixels, convert using your window size and pixel size (or your renderer’s scaling) when needed.
  • Focus changes: on focus loss, held buttons and drag state are cleared and mouse capture is released to avoid “stuck” inputs. On focus gain, per-frame deltas reset; re-apply any custom cursor/confine/relative mode settings as needed.
  • Double-click tuning: time/radius changes affect future click detection. Apply your preferred values during initialization for consistent behavior.

Troubleshooting

  • Double-clicks not recognized: ensure you set time/radius early; verify your game loop reads per-frame edges each Update.
  • Positions don’t match pixels: remember X/Y are logical coordinates. Map to render pixels using your window/render target scaling.
  • Input stuck after alt-tab: focus loss clears held state by design. Ensure your input handling doesn’t assume a button remains down across focus changes.
