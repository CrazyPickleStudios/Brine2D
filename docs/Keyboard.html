Keyboard input

Brine2D exposes keyboard state via IKeyboard (and on some hosts, the equivalent IInput surface). It provides per-frame key edges and “is down” queries. Desktop hosts also offer modifier/chord helpers and text input (IME).

Quick start

// Access the keyboard from your engine context (e.g., in Update)
var kb = context as IKeyboard // if your host exposes IKeyboard
         ?? (context as IEngineContext)?.Input; // or IInput on desktop host

if (kb.WasKeyPressed(Key.Space))
{
    // Fired on the frame Space transitioned up -> down
}

if (kb.IsKeyDown(Key.Left) || kb.IsKeyDown(Key.A))
{
    // Held this frame
}

Core API

  • bool IsKeyDown(Key key) — true while the key is held.
  • bool WasKeyPressed(Key key) — true only on the frame up → down.
  • bool WasKeyReleased(Key key) — true only on the frame down → up.

Key identifiers are defined in Key, covering navigation, letters/digits, function keys, modifiers (left/right), numpad and common media keys.

Modifiers and chords (desktop host)

The SDL desktop host exposes modifier state and chord helpers on SdlHost:

  • KeyboardModifiers Modifiers — current modifiers (Shift/Ctrl/Alt/Super, side-specific, locks).
  • bool IsChordDown(Key key, KeyboardModifiers required)
  • bool WasChordPressed(Key key, KeyboardModifiers required)
  • bool WasShortcutPressed(Key key) — Ctrl on Windows/Linux, Cmd on macOS.
  • static string FormatChord(KeyboardModifiers mods, Key key) — e.g., “Ctrl+Shift+S”.
if (host.WasChordPressed(Key.Enter, KeyboardModifiers.Alt))
    ToggleFullscreen();

if (host.WasShortcutPressed(Key.S))
    SaveGame();

var label = SdlHost.FormatChord(KeyboardModifiers.Control | KeyboardModifiers.Shift, Key.S);

Text input and IME (desktop host)

For text entry (names, chat, console) use text input and IME composition:

  • void StartTextInput(Rectangle? imeRect = null) — begin text input. Optional caret area in window coords.
  • void StartTextInput(Rectangle imeRect, int caretOffset) — explicit caret offset (relative to rect.x).
  • void StopTextInput() — end text input.
  • IReadOnlyList<string> TypedText — committed UTF-8 text chunks this frame.
  • string CompositionText, int CompositionStart, int CompositionLength, bool IsComposing — IME composition info.
// Start with a caret box at (x,y,w,h), caret at end
host.StartTextInput(new Rectangle(x, y, w, h));

// Per frame: consume typed characters
foreach (var chunk in host.TypedText)
    textBuffer.Append(chunk);

// IME composition (show preedit with underline/selection)
if (host.IsComposing)
    DrawComposition(host.CompositionText, host.CompositionStart, host.CompositionLength);

// Done
host.StopTextInput();

Repeat and advanced behavior (desktop host)

  • SetSuppressKeyRepeat(bool) — ignore OS key repeat for edge generation (default: true).
  • SetKeyRepeatGeneratesPressed(bool) — treat repeats as WasKeyPressed when enabled (menus/held navigation).

Rebinding and chords (optional utility)

The core includes a small binding utility for actions with keyboard chords:

  • KeyChord — a key + required modifiers.
  • KeyChordFormat — format/parse “Ctrl+Shift+S”, supports aliases like “PgUp”, “Num5”, “KP_1”.
  • KeyBindings<TAction> — map actions to one or more chords; query WasPressed/IsDown/WasReleased.
var bindings = new KeyBindings<GameAction>();
bindings.Bind(GameAction.ToggleFullscreen, Key.Enter, KeyboardModifiers.Alt);
bindings.Bind(GameAction.Save, Key.S, KeyboardModifiers.Control);

if (bindings.WasPressed(GameAction.Save, host, host.Modifiers))
    SaveGame();

Examples

Discrete movement and actions

var kb = (IKeyboard)host;
var move = new Vector2(0, 0);
if (kb.IsKeyDown(Key.W) || kb.IsKeyDown(Key.Up)) move.Y -= 1;
if (kb.IsKeyDown(Key.S) || kb.IsKeyDown(Key.Down)) move.Y += 1;
if (kb.IsKeyDown(Key.A) || kb.IsKeyDown(Key.Left)) move.X -= 1;
if (kb.IsKeyDown(Key.D) || kb.IsKeyDown(Key.Right)) move.X += 1;

if (kb.WasKeyPressed(Key.Space))
    Jump();

Shortcuts and platform-friendly labels

bool savePressed = host.WasShortcutPressed(Key.S);
string label = SdlHost.FormatChord(
    OperatingSystem.IsMacOS() ? KeyboardModifiers.Super : KeyboardModifiers.Control,
    Key.S);

Behavior notes

  • Per-frame edges: call queries during your Update. Edge state resets each frame.
  • Focus changes: on focus loss, held keys are cleared to prevent “stuck” input.
  • Modifiers and locks: desktop host synchronizes with the OS (e.g., Caps/Num Lock) for accurate chord detection.
  • Numpad vs top-row digits: both are exposed (e.g., D1 vs Numpad1) and can be bound independently.

Troubleshooting

  • Edges not firing: ensure you’re not discarding Update frames and that you query within the frame they occur.
  • Shortcuts inconsistent across OS: prefer WasShortcutPressed(key) to map Ctrl/Cmd automatically.
  • IME caret placement: StartTextInput(rect, caretOffset) expects window coordinates; pass the caret offset relative to rect.x.
