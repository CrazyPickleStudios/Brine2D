Text & fonts

Brine2D exposes text rendering through the IFont interface. Internally this is backed by one or more platform fonts (SpriteFont) combined via a composite for fallback (CompositeFont). User code only needs IFont. Features include glyph caching into a GPU atlas, measurement, word wrapping, truncation with ellipsis, justification (space distribution), selection highlighting, caret positioning, grapheme-aware layout, and style spans.

Quick start

// Load a font (single or fallback chain) via content:
// Descriptor: FamilyA+FamilyB@Size[optional $MissingPrimary].spritefont
var font = context.Content.Load<IFont>("JetBrainsMono+NotoEmoji@16.spritefont");

// Simple draw
font.DrawString(context.Sprites, "Hello Brine2D!", 32, 48, Color.White);

// Wrapped draw (max width, left align)
font.DrawStringWrapped(context.Sprites, "Long paragraph ...", 32, 120, 420, TextAlign.Left, Color.White);

// Measure text (width,height in unscaled pixels)
var (w, h) = font.Measure("Sample\nLines");

// Build a layout (for selection / caret / decorations / custom scale)
var layout = font.BuildLayout(
    "Editable text sample",
    originX: 32,
    originY: 200,
    maxWidth: 480,
    align: TextAlign.Left,
    scale: 1f);

// Draw layout
font.DrawLayout(context.Sprites, layout, Color.White);

// Draw caret (index is UTF-16 position in source string)
font.DrawCaret(context.Sprites, layout, index: 5, Color.Yellow, thickness: 2);

Font descriptor (.spritefont)

  • Pattern: FamilyA+FamilyB+FamilyC@Size[$HexCodepoint].spritefont
  • Fallback chain: families are checked in order for missing glyphs.
  • Missing glyph override: optional primary sentinel after $ (e.g. $FFFD).
  • Examples:
    • JetBrainsMono@16.spritefont
    • JetBrainsMono+NotoEmoji@16$FFFD.spritefont
  • Raw font shortcuts: size suffix on TTF/OTF/TTC is supported by the loader, e.g. SomeFont.ttf@18.

File formats & loading

  • Containers: TrueType (.ttf), OpenType (.otf), and TrueType Collections (.ttc).
  • Naming probes (per family, for a given size N):
    • Family.ttf@N, Family.otf@N, Family.ttc@N
    • Family@N.ttf, Family@N.otf, Family@N.ttc
  • Place assets under Content/ or Assets/ according to your providers.

Core API (IFont)

  • Measure(string), MeasureWrapped(string, maxWidth) (returns width,height).
  • DrawString(renderer, text, x, y, color, scale=1f)
  • DrawStringWrapped(renderer, text, x, y, maxWidth, align, color, scale=1f)
  • BuildLayout(text, originX, originY, maxWidth?, align, scale, styleSpans?, selectionStart, selectionLength, selectionColor?) → returns TextLayout (glyph positions, lines, selection rectangles, caret stops).
  • DrawLayout(renderer, layout, baseColor, overrideScale?)
  • DrawCaret(renderer, layout, index, color, thickness=1)
  • PrewarmAscii(), Prewarm(string), PrewarmRange(int start, int end) (populate atlas ahead of time).
  • ClearCache() (reset glyph + measure caches; triggers re-rasterization).

Layout features

  • Word wrapping: space, tab, hyphen, non‑breaking space break points.
  • Justification: evenly distributes extra width across internal spaces (not last line).
  • Truncation: optional width limit with ellipsis (defaults to …).
  • Selection highlighting: rectangular background for selected range.
  • Caret stops: positions for editing navigation (grapheme-aware if configured).
  • Style spans: per-range color + text decorations (underline / strikethrough).
  • Grapheme clusters: optional cluster mode joins combining marks / surrogate pairs for caret placement.
  • Scale: uniform scaling of line height and glyph geometry (integer output size clamped ≥ 1).

Truncation & ellipsis

var options = new BuildLayoutOptions(
    maxWidth: 320,
    align: TextAlign.Left,
    scale: 1f,
    truncateWidth: 300,
    ellipsis: "...");

var layout = font.BuildLayout("Very long line that must fit", 24, 24, options);
font.DrawLayout(context.Sprites, layout, Color.White);

Justification

var options = new BuildLayoutOptions(
    maxWidth: 480,
    align: TextAlign.Left,
    scale: 1f,
    justify: true);

var layout = font.BuildLayout("Multiple words per line for justification demo", 32, 320, options);
font.DrawLayout(context.Sprites, layout, Color.White);

Grapheme-aware caret

Enable when you handle user text editing with combining marks, emoji sequences, or surrogate pairs:

var options = new BuildLayoutOptions(maxWidth: 520, align: TextAlign.Left, scale: 1f, useGraphemeClusters: true);
var layout = font.BuildLayout("Emoji 👍🏽 test", 32, 400, options);
font.DrawLayout(context.Sprites, layout, Color.White);
font.DrawCaret(context.Sprites, layout, index: 7, Color.Cyan); // caret aligns at cluster boundary

Prewarm (performance)

  • Why: Avoid runtime stalls fetching and uploading glyph bitmaps during gameplay.
  • Use: 
    // Before heavy UI / scene load
font.PrewarmAscii();
font.Prewarm("HUD labels 0123456789 Score Lives Pause Resume");
font.PrewarmRange(0x2190, 0x21FF); // arrows block

Threading & lifetime

  • Single-threaded usage: SpriteFont mutates a GPU atlas and caches. Call Prewarm, Measure, BuildLayout, and draw methods on the font’s creator/main thread.
  • Cache resets: ClearCache() and font disposal should also happen on the creator thread.
  • Reuse: Reuse loaded IFont instances; each instance owns its own atlas pages.

Font fallback

When using a descriptor with multiple families, missing glyphs are resolved by scanning the remaining fonts. Results are cached per codepoint (fast after first lookup).

  • Primary first; counts fallback hits internally (diagnostics).
  • Unsupported glyphs default to a configured missing glyph sentinel (e.g. ? or U+FFFD).

Current limitations

  • Approximate kerning using measured pair widths (not full GPOS). Adequate for many Latin fonts but not typographically perfect.
  • No advanced OpenType shaping yet (ligatures, mark positioning) — reserved extension via ITextShaper.
  • RTL / bidirectional scripts handled only at basic codepoint order for now (no reordering / shaping).

Future shaping (HarfBuzz integration)

A pluggable shaper (ITextShaper) is present; advanced mode would supply:

  • Proper glyph substitution (GSUB), positioning (GPOS), ligatures.
  • Script & direction runs (Arabic, Devanagari, mixed RTL/LTR).
  • Accurate caret movement over clusters and combined marks.

Troubleshooting

  • Text width seems off: Ensure you measured with the same ExtraLineSpacing, TabSpaces, and scale; caches key off these values.
  • Glyphs pop in late: Prewarm glyph ranges used in early UI; large first-frame stalls indicate rasterization on demand.
  • Missing characters: Add an appropriate fallback font (e.g. Emoji or CJK) by listing multiple families in the descriptor.
  • Layout perf: Reuse layouts if text does not change; BuildLayout is more expensive than simple draws.
  • Couldn’t load a family: Verify the loader’s probe patterns (e.g. Family.ttf@16 and Family@16.ttf) exist under your content roots.

Best practices

  • Centralize font loading early; reuse IFont instances (glyph atlas is per font).
  • Use BuildLayout only when you need selection, caret, truncation or styling; simple labels can call DrawString.
  • Prewarm big UI sets in a loading phase; keep runtime glyph uploads small.
  • Plan shaping integration before large localization effort for complex scripts.

Quick reference cheatsheet

// Measure
var (mw, mh) = font.Measure("Title");

// Draw unwrapped
font.DrawString(context.Sprites, "Score: 123", 16, 16, Color.White);

// Draw wrapped & centered
font.DrawStringWrapped(context.Sprites, paragraph, 40, 200, 360, TextAlign.Center, Color.White);

// Build styled layout
var spans = new [] { new StyleSpan(0, 5, Color.Gold, decorations: TextDecoration.Underline) };
var layout = font.BuildLayout("TitleLine", 40, 300, maxWidth: null, align: TextAlign.Left, scale: 1f, styleSpans: spans);
font.DrawLayout(context.Sprites, layout, Color.White);

// Caret
font.DrawCaret(context.Sprites, layout, 3, Color.Cyan);

// Truncated
var opts = new BuildLayoutOptions(maxWidth: 420, align: TextAlign.Left, scale: 1f, truncateWidth: 400, ellipsis: "…");
var truncated = font.BuildLayout(longText, 32, 360, opts);
font.DrawLayout(context.Sprites, truncated, Color.White);

Note about the built-in DebugOverlay

The DebugOverlay renders its text via a direct SDL_ttf path for robustness and minimal dependencies. This is separate from IFont and intended to keep the overlay functional even during engine startup/shutdown. Game/UI text should use IFont.
